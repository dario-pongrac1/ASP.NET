3. `OrderBy(o => o.ScheduledAt)` sortira te filtrirane narudžbe od najranijeg prema kasnijem terminu.
4. `FirstOrDefault()` uzima prvi element iz tako sortirane kolekcije.
5. Ako nema niti jedne takve narudžbe, vraća `null`.

Zašto rezultat ispada `SO-2026-003`:
- u seedu mehaničar 2 ima jednu narudžbu
- zato filtriranje ostavi tu jednu
- `FirstOrDefault()` vrati baš nju

Bitna razlika:
- `FirstOrDefault` ne traži “jedinstveni” element
- uzima prvi koji odgovara uvjetu ili `null` ako ga nema

---

**3. SingleOrDefault**
U liniji:
Program.cs

```csharp
var orderByNumber = context.ServiceOrders
    .SingleOrDefault(o => o.OrderNumber == "SO-2026-002");
```

Ovo traži točno jednu narudžbu s brojem `SO-2026-002`.

Kako radi:
1. Krene kroz sve narudžbe.
2. Gleda koje imaju `OrderNumber == "SO-2026-002"`.
3. Ako nađe točno jednu, vrati je.
4. Ako ne nađe nijednu, vrati `null`.
5. Ako nađe više od jedne, baca iznimku.

Zašto je ovdje dobar primjer:
- `OrderNumber` bi u stvarnom sustavu trebao biti jedinstven
- zato je `SingleOrDefault` logičan izbor

Zašto je rezultat `InProgress`:
- u seedu narudžba `SO-2026-002` ima status `InProgress`
- zato ispisuje taj status

Bitna razlika u odnosu na `FirstOrDefault`:
- `FirstOrDefault` ne mari ako ima više podudaranja
- `SingleOrDefault` očekuje najviše jedan rezultat

---

**4. Where + OrderBy + ToList**
U liniji:
Program.cs

```csharp
var ordersInPeriod = context.ServiceOrders
    .Where(o => o.ScheduledAt >= new DateTime(2026, 4, 2) && o.ScheduledAt <= new DateTime(2026, 4, 3, 23, 59, 59))
    .OrderBy(o => o.ScheduledAt)
    .ToList();
```

Ovo vraća sve narudžbe koje su zakazane između 2.4.2026. i kraja 3.4.2026., sortirane po datumu.

Kako ide:
1. `Where(...)` filtrira narudžbe po datumu.
2. Uvjet provjerava je li `ScheduledAt` unutar raspona.
3. `OrderBy(o => o.ScheduledAt)` ih sortira od ranijeg prema kasnijem.
4. `ToList()` pretvara rezultat u konkretnu `List<ServiceOrder>`.

Zašto je tu `ToList()` važan:
- bez njega dobio biš samo odgođeni LINQ rezultat
- s `ToList()` se izvršava i materializira u listu

Zašto rezultat ispada 3:
- u seedu sve 3 narudžbe upadaju u taj vremenski raspon
- zato `ordersInPeriod.Count` vrati 3

**5. Select + Count + OrderByDescending**
U liniji:
Program.cs

```csharp
var activePerMechanic = context.Mechanics
    .Select(m => new
    {
        Mechanic = $"{m.FirstName} {m.LastName}",
        ActiveOrders = m.ServiceOrders.Count(o => o.Status == OrderStatus.Scheduled || o.Status == OrderStatus.InProgress)
    })
    .OrderByDescending(x => x.ActiveOrders)
    .ToList();
```

Ovo radi projekciju mehaničara u anonimni objekt koji sadrži ime i broj aktivnih narudžbi.

Kako radi:
1. `context.Mechanics` uzme sve mehaničare.
2. `Select(...)` pretvara svaki objekt `Mechanic` u novi oblik.
3. Novi oblik ima:
   - puno ime
   - broj aktivnih narudžbi
4. `ActiveOrders` se računa tako da se u `m.ServiceOrders` broje samo narudžbe sa statusom `Scheduled` ili `InProgress`.
5. `OrderByDescending(x => x.ActiveOrders)` sortira mehaničare od najzaposlenijeg prema manje zaposlenom.
6. `ToList()` materijalizira rezultat.

Zašto je ovo dobar primjer:
- pokazuje `Select`, dakle projekciju
- pokazuje i rad s navigacijskim kolekcijama
- pokazuje agregat unutar projekcije

Zašto svi imaju po 1:
- u seedu svaki mehaničar ima jednu vezanu narudžbu
- sve su te narudžbe aktivne jer su `Scheduled` ili `InProgress`

**6. GroupBy + Select + Count + OrderByDescending + Take**
U liniji:
Program.cs

```csharp
var topServices = context.OrderLines
    .GroupBy(line => line.ServiceItem!.Name)
    .Select(g => new { ServiceName = g.Key, UsageCount = g.Count() })
    .OrderByDescending(x => x.UsageCount)
    .ThenBy(x => x.ServiceName)
    .Take(3)
    .ToList();
```

Ovo grupira stavke narudžbi po nazivu usluge i pokazuje koje su usluge najčešće korištene.

Kako radi:
1. `context.OrderLines` uzme sve stavke narudžbi.
2. `GroupBy(line => line.ServiceItem!.Name)` grupira ih po nazivu usluge.
3. Svaka grupa predstavlja jednu uslugu, npr. “OBD dijagnostika”, “Zamjena ulja”.
4. `Select(g => new { ... })` pretvara grupu u objekt s:
   - nazivom usluge
   - brojem pojavljivanja
5. `Count()` na grupi vraća koliko stavki pripada toj usluzi.
6. `OrderByDescending(x => x.UsageCount)` sortira najčešće prema rjeđe korištenima.
7. `ThenBy(x => x.ServiceName)` služi kao sekundarno sortiranje ako je broj jednak.
8. `Take(3)` uzima samo prva 3 rezultata.
9. `ToList()` izvršava sve i vraća listu.

Zašto je ovdje `line.ServiceItem!.Name`:
- `OrderLine` nema vlastito ime usluge, nego preko navigacije dolazi do `ServiceItem`
- `!` kaže compileru da znaš da nije `null`, jer je u seedu povezan

Zašto su rezultati baš takvi:
- u seedu neke usluge se ponavljaju više puta kroz različite narudžbe
- zato `Count()` po grupama pokazuje koliko je puta usluga iskorištena

## Što je async/await u C#
`async/await` je način da napišeš neblokirajući kod koji čeka spor posao (npr. baza, API, file I/O), ali bez blokiranja dretve koja je pozvala metodu.

Ključna ideja:
1. `async` označi metodu koja može imati `await`.
2. `await` kaže: “stani ovdje dok se Task ne završi, ali nemoj blokirati dretvu”.
3. Metoda uglavnom vraća `Task` ili `Task<T>`.

---

## Zašto je to važno
Bez async:
1. dretva čeka i “stoji” dok se posao završi
2. slabija skalabilnost (posebno na web serveru)

S async:
1. dok čekaš I/O, dretva se može osloboditi za druge zahtjeve
2. bolja responzivnost i skalabilnost

Jedna rečenica za profesora:
“Async/await ne ubrzava nužno sam posao, ali povećava propusnost jer dretve nisu blokirane tijekom čekanja.”

---

## Što je Task
`Task` predstavlja “rad koji traje” i završit će kasnije.

Tipovi:
1. `Task` -> nema povratne vrijednosti
2. `Task<T>` -> vraća rezultat tipa `T`

Primjer:
```csharp
async Task<int> GetCountAsync()
{
    await Task.Delay(100);
    return 5;
}
```

---

## Razlika: sinkrono vs asinkrono čekanje

Sinkrono čekanje:
```csharp
Thread.Sleep(1000);
```
- blokira dretvu 1 sekundu

Asinkrono čekanje:
```csharp
await Task.Delay(1000);
```
- ne blokira dretvu; metoda se “pauzira”, a nastavi kad Task završi

---

## Kako teče izvršavanje s await
Kad dođe do `await someTask`:
1. ako `someTask` nije gotov, metoda se privremeno prekine
2. kontrola se vrati pozivatelju
3. kad Task završi, metoda nastavlja od linije iza `await`

To zovemo continuation (nastavak izvođenja).

---

## Pravila koja trebaš znati za usmeno

1. “Async all the way”
Ako je metoda async, i metoda koja je zove često treba biti async.

2. Izbjegavaj `.Result` i `.Wait()`
Mogu uzrokovati blokiranje i deadlock u nekim okruženjima.

3. Potpis metode:
- koristi `Task` ili `Task<T>`
- `async void` samo za event handlere

4. CPU-bound vs I/O-bound:
- I/O-bound: `await` (baza, mreža, disk)
- CPU-bound: eventualno `Task.Run(...)` ako želiš odvojiti računanje

---

## Paralelizacija s Task.WhenAll
Ako imaš više neovisnih async poziva, ne moraš ih čekati jedan po jedan.

Sekvencijalno (sporije):
```csharp
var a = await AAsync();
var b = await BAsync();
```

Paralelno:
```csharp
var taskA = AAsync();
var taskB = BAsync();
await Task.WhenAll(taskA, taskB);
var a = taskA.Result;
var b = taskB.Result;
```

Za objasniti:
“`Task.WhenAll` pokreće više async operacija paralelno i čeka da sve završe.”

2. dohvat zauzeća mehaničara

To su 3 neovisna upita i mogu ići paralelno s `Task.WhenAll`.
